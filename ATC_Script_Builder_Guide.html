01 Opening the App
💻On a Computer▼
01Locate ATC_Script_Builder.html on your computer or download it from the company cloud.
02Double-click the file — it opens in your default browser. Use Chrome or Edge for best results.
03If a security warning appears, click Allow or Run Anyway.
04No internet required once the file is on your computer — it runs entirely offline.
⚠️ Avoid Internet Explorer
The app will not work in Internet Explorer. Use Chrome or Edge only.
📱On iPad▼
01Download ATC_Script_Builder.html from the company cloud to your iPad Files app.
02Tap the file in Files — it opens in Safari automatically.
03To load a script tap 📂 Load and select the .json file from Files.
⚠️ Use Safari on iPad
Chrome on iPad may block local file access. Always use Safari.

03 Building a Script
📋Step-by-step walkthrough▼
01Script Header — fill in Exercise Title, Date, Exercise Ref and Notes. Notes field: each line becomes a red bullet point in the script.
02Frequencies — toggle on the positions you need (TWR, GND, DEL etc.) and type the frequency number. These populate the frequency dropdown on every entry.
03Weather — fill in Station (dropdown), Obs Time, ATIS, Approach Type, Runway, Wind, Visibility, Temp/Dew Point, QNH, Cloud, Surface, NOTAM and 2000ft Wind.
04New Aircraft Entry — select Entry Type and Frequency at the top. Fill in Callsign, select Aircraft Type from the dropdown (194 aircraft), enter Transponder and Flight Level separately, then Route, Times, Waypoints, Pilot Speech and Sim Notes.
05Click ADD TO SCRIPT ▶ — entry auto-sorts by time. Repeat for each aircraft.
06Use ↑ ↓ arrows to manually reorder. Click ✏️ to edit an entry. Click ✕ to delete.
07Click 💾 Save to download the .json file. Upload to the company cloud.
💡 Keyboard Navigation
Press Enter or ↓ to move to the next field. Press ↑ to go back. In multi-line fields (instructions, route, aircraft type) press ↓ to move to the next field.
✏️Editing an existing entry▼
01Click the ✏️ button on any entry row in the builder.
02The form fills with that entry's data. For IFR/VFR departures, all rows for that callsign load together so you can add a missing time or change any call.
03Make your changes. The ADD button turns orange and says UPDATE ENTRY ✔.
04Click UPDATE ENTRY ✔ — changes save and the script re-sorts by time automatically.
05Click ✕ Cancel Edit to discard changes.
04 Frequencies
📻Setting up frequencies▼
01In the Script Header, find the Frequencies section with pill buttons: TWR · GND · DEL · APP · DEP · INFO · APRON.
02Click a pill to toggle it on — it turns blue and a number input appears.
03Type the frequency number (e.g. 118.3). It immediately appears in all entry frequency dropdowns.
04Only frequencies with a number entered show in the view script header and in the entry dropdown.
05Click the pill again to turn it off and remove it from the dropdowns.
✅ Set once per script
Define your frequencies at the start and every entry dropdown is pre-populated. Frequencies save and load with the .json file.
05 IFR & VFR Departure — 3 Calls
⚡IFR Departure — Initial / Start / Request Taxi▼
①Initial Call Time — enter the time and the full pilot speech in the Pilot Speech field. Add sim notes below the time if needed.
②Start Time — generates a "Start" row automatically. Leave blank to skip. Add sim notes if needed.
③Request Taxi Time — generates "Request Taxi" (overrideable in the taxi speech field). Add sim notes if needed.
✈️ Requires Pushback?
Tick the Requires Pushback checkbox to add Pushback and Start time fields.

Same call? Tick the Same call checkbox next to Pushback Time to generate a single "Request Start and Pushback" row instead of two separate rows. You can also skip the Initial Call Time entirely and go straight to any other time if the flight is already in progress.
⚡VFR Departure — Initial / Request Taxi▼
①Initial Call Time — full speech in the Pilot Speech field (e.g. "4 on board for Motiti, H1013, request Taxi"). Add sim notes if needed.
②Request Taxi Time — generates "Request Taxi" automatically. Add sim notes if needed.
💡 All times optional
Leave any time blank to skip that row entirely.
06 Saving, Loading & Editing
💾Saving a script▼
01Click 💾 Save — a .json file downloads named after the Exercise Title field.
02Upload the .json to the company cloud for pilots to access.
⛔ No auto-save
The app does NOT auto-save. Always click Save before closing the tab — unsaved work will be lost.
📂Loading a script▼
01Download the .json from the company cloud.
02Click 📂 Load and select the file — the entire script loads and opens in View Script mode.
03To edit, click ✏️ Edit Script to return to the builder.
⚠️ Wrong file?
Only .json files saved by this app will load. Do not try to load a PDF, Word doc or other format.
07 iPad Workflow & PDF Annotation
📱Annotating with Apple Pencil▼
01Download the .json from the company cloud to your iPad.
02Open the app in Safari, tap 📂 Load and select the script.
03Tap 📄 Export PDF — a new tab opens with the clean script.
04Tap the Share button → Save to Files → saves as PDF.
05Open in GoodNotes or Notability and annotate with Apple Pencil.
✅ Best iPad workflow
Keep the .html file saved to your iPad permanently. Download a fresh .json before each simulation, load it, export PDF, annotate.
08 Exporting & Printing
🖨Printing with colours▼
01Click the Print tab then 🖨 Print.
02In the print dialog, enable Background graphics — otherwise row colours won't print.
03Set orientation to Landscape for best fit.
⚠️ Colours not printing?
Chrome: More settings → enable "Background graphics". Edge: More settings → enable "Background colors and images".
📄Exporting as PDF▼
01Click 📄 Export PDF from View Script or Print.
02Computer: In the print dialog choose Save as PDF.
03iPad: Share button → Save to Files.
⚠️ PDF export opens blank tab?
Your browser is blocking pop-ups. Look for the blocked pop-up icon in the address bar and click Allow.
09 Common Problems & Fixes
🚨App won't open / blank screen▼
01Right-click the file → Open with → Chrome or Edge.
02If a security warning shows, click Allow or Run Anyway.
03Press F5 to refresh.
🚨Entries disappeared after closing the tab▼
⛔ No auto-save
The app does not auto-save. Always click 💾 Save before closing. There is no way to recover unsaved work.
🚨Load button shows error / invalid file▼
01Make sure you are loading a .json file saved by this app — not a PDF or Word doc.
02Try downloading the file from the cloud again — it may be partially downloaded.
🚨Entries not sorting in the right order▼
01Make sure times are entered as 4-digit numbers — e.g. 2003 not 20:03.
02Use the ↑ ↓ arrows to manually adjust order if needed.
🚨Frequency dropdown is empty▼
01Go to the Script Header and toggle on the frequency positions you need (TWR, GND etc.).
02Type a number next to each active position — only frequencies with a number appear in the dropdown.
🚨ATIS / QNH not showing in footer▼

The ATIS and QNH footer at the bottom of the script only shows if you have filled in the ATIS and QNH fields in the Weather Information section of the builder.

🚨Fonts look plain / different▼

The app uses Google Fonts which require internet to load. Offline the app falls back to system fonts — everything still works correctly, it just looks slightly different.

10 Editing the Code Yourself
🔧How to open and edit the file▼
01Always back up first — copy the file and rename it (e.g. ATC_Script_Builder_backup.html) before making any changes.
02Right-click ATC_Script_Builder.html → Open with → Notepad (Windows) or TextEdit (Mac).
03Press Ctrl+F (Windows) or Cmd+F (Mac) to search for the section you want to edit.
04Make your change, then Save (Ctrl+S). Refresh the browser to see the result.
🔍 Section markers
Search for these labels to jump to the right place:

SECTION 1 — COLOUR CODES → change entry type colours
SECTION 2 — ENTRY TYPES → rename entry type labels
SECTION 3 — AIRPORT LIST → add/remove/rename airports
SECTION 4 — DEFAULT WEATHER → change default weather values
SECTION 9 — FREQUENCY MANAGEMENT → change default frequency positions
✈️Adding or changing an airport▼

Search for SECTION 3 — AIRPORT LIST to find the airport list. Each entry looks like this:

{ code: 'NZTG', name: 'NZTG — Tauranga' },

To add: copy any line, change the code and name, paste it in the right group.

To remove: delete the whole line.

To rename: change the text after the — inside the name field.

⚠️ Apostrophes in names
If the name contains an apostrophe (e.g. Hawke's Bay) you must write it as Hawke\'s Bay with a backslash, otherwise the file will break.
✈️Adding a new aircraft to the dropdown▼

Search for SECTION 4 — AIRCRAFT LIST to find the aircraft list. Each entry is just the model name in single quotes followed by a comma:

'DH8C','DH8D','DH82',

To add: find the right alphabetical position and insert the new model name:

'DH8C','DH8D','DH82','DHC3',  ← correct

To remove: delete the entry including its comma.

⚠️ Keep the comma
Every entry must end with a comma. Missing commas will break the app.
🎨Changing a colour▼

Search for SECTION 1 — COLOUR CODES. You'll see entries like:

ifr_dep: { bg: '#8DB3E2', light: '#C6D9F1', ... },

bg = the darker shade (callsign/time cell). light = the lighter shade (instructions row).

Change the hex colour code — e.g. replace '#8DB3E2' with '#aabbcc'. Keep the quotes and the hash symbol.

11 Common Coding Errors & How to Fix Them
🚨App goes blank after editing the file▼

This means there is a syntax error in the JavaScript. The most common causes:

1Missing comma after an airport entry. Every line in the airport list must end with a comma:
{ code: 'NZTG', name: 'NZTG — Tauranga' },  ← correct{ code: 'NZTG', name: 'NZTG — Tauranga' }   ← missing comma, will break
2Unescaped apostrophe in a name. If a name has an apostrophe you must escape it:
name: 'Hawke\'s Bay'   ← correctname: 'Hawke's Bay'   ← will break
3Mismatched quotes — make sure every opening ' has a closing '.
4Accidental deletion of a ]; or }; that closes a section.
✅ Fix: restore from backup
If the app goes blank, close the file without saving and restore from your backup copy. Always keep a backup before editing.
🚨New airport not showing in dropdown▼
01Check the line ends with a comma: { code: 'NZXX', name: 'NZXX — Name' },
02Check the code and name are inside single quotes ' — not double quotes ".
03Save the file and hard refresh the browser: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac).
🚨Changes not showing after saving the file▼
01The browser may be showing a cached version. Press Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac) to force a full refresh.
02Make sure you saved the correct file — check you edited ATC_Script_Builder.html and not the backup copy.
🚨Uncaught SyntaxError in browser console▼

If you want to find exactly where the error is:

01Press F12 to open browser Developer Tools.
02Click the Console tab — the error will show with a line number.
03Go to that line number in Notepad and look for a missing comma, quote or bracket near that area.
04If you can't find it, restore from your backup and try the edit again more carefully.
💡 Line numbers in Notepad
In Notepad on Windows: Format menu → check Word Wrap is OFF, then look at the bottom of the window — it shows the current line number as you click around.
🚨Colour changed but looks the same▼
01Make sure you changed both bg and light if you want both rows to change.
02Hard refresh the browser: Ctrl+Shift+R.
03Check the hex code starts with # and is 6 characters, e.g. #8DB3E2 — not 8DB3E2 or #8db.
